CAEXDocument Methods

AutomationML 2.1 API

The CAEXDocument type exposes the following members.

Methods
  NameDescription
AMLSchemaManager(XElement)
Gets the schema manager for the CAEX document which contains the provided XML node.
AMLSchemaManager(CAEXWrapper)
Gets the schema manager for the CAEX document which contains the provided CAEX object.
Document(XElement)
Gets the CAEX document which contains the specified XML node. If the node is not contained in an XML document, no CAEX document is returned.
Document(CAEXWrapper)
Gets the CAEX document which contains the specified CAEX object. If the CAEX object has not been inserted into a CAEX document, no CAEX document is returned.
LoadFromBinary
Loading a CAEX file from the specified Byte-Array.
LoadFromFile
Create a CAEX document with content, loaded from a file. The schema version of the CAEX document is set accordingly to the SchemaVersion-Attribute of the CAEXFile object.
LoadFromStream
Create a CAEX document with content, loaded from a stream. The schema version of the CAEX document is set accordingly to the SchemaVersion-Attribute of the CAEXFile object.
New_CAEXDocument
Creates a new empty CAEX document containing only the CAEXFile Element as the root element. The CAEX document is created, using the default schema version DefaultSchema.
New_CAEXDocument(CAEXDocumentCAEXSchema)
Creates a new empty CAEX document containing only the CAEXFile element as the root element. The CAEX document is created using the specified schema version. The specified schema version defines the applicable editing methods.
SaveToFile
Saves the CAEX document to the specified file path.
SaveToStream
Saves the CAEX document to the specified file path.
Validate
Checks if the document is schema compliant.
XName
Gets the correct XName for the provided CAEX element name (Tag name).The XName contains a namespace, if the assigned schema requires a namespace. If no namespace is defined for the assigned schema (as in CAEX 2.15), no namespace is added to the XName. The XName has to be used for all Xml.Linq queries, used to access objects with the CAEX element name.

ClassIsDerivedFromT
Determines, if the class, specified with the given class path is directly or indirectly (across multiple generations) a descendant of the class or identical to it, which is uniquely identified with the specified baseClass path.
